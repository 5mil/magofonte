◆ What is Lancia?
◆ Purpose
Lancia is your one-page command centre for spinning up, monitoring, and tearing down MagoFonte mining pool instances on free-tier cloud infrastructure — no local server required. Each instance runs the full MagoFonte stack: Stratum server, API, wallet manager, and block broadcaster.
◆ How it works
You fill in the Launch form below. Lancia calls the MagoFonte REST API (/api/v1/lancia/instances) which provisions a container on your chosen provider, injects your config as environment variables, and returns a live instance ID. The Fleet panel then shows its status in real time.
◆ Pool economics
Every instance runs PPLNS payouts. Pool fee is fixed at 2% of every block reward — this funds infrastructure and development. Miners keep the remaining 98%. Minimum payout is configurable per instance (default 1 DGB).
◆ Prerequisites
A MagoFonte API endpoint running (local or hosted) — this page calls it
A DGB reward wallet address to receive block rewards
A free account on at least one supported provider (Fly.io, Railway, Render, or Oracle)
Your provider API token or deploy hook pasted into the form

◆ Reliability & Auto-Restart
Auto-Restart on Crash ?
Instance restarts automatically if it exits unexpectedly. Recommended for production.
⚠  Crash-loop guard: If an instance restarts more than 3 times within 5 minutes, Lancia will halt further restarts and flag it as Crash-Halted in the fleet. You will need to manually investigate logs and click Resume to re-enable auto-restart. This prevents runaway billing and masks silent misconfigurations.
◆ Advanced — Environment Overrides
One KEY=VALUE per line. These are passed as container environment variables and override image defaults.
❱ Launch Log ❰
Real-time activity log for this session. Shows API calls, instance state changes, errors, and crash-guard events. Color key: green = success, gold = warning, red = error, dim gold = info.
