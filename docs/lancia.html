◆ What is Lancia?
◆ Purpose
Lancia is your one-page command centre for spinning up, monitoring, and tearing down MagoFonte mining pool instances on free-tier cloud infrastructure — no local server required. Each instance runs the full MagoFonte stack: Stratum server, API, wallet manager, and block broadcaster.
◆ How it works
Fill in the Launch form below. Lancia calls the MagoFonte REST API (/api/v1/lancia/instances) which provisions a container on your chosen provider, injects your config as environment variables, and returns a live instance ID. The Fleet panel shows its status in real time.
◆ Pool economics
Every instance runs PPLNS payouts. Pool fee is fixed at 2% of every block reward — this funds infrastructure and development. Miners keep the remaining 98%. Minimum payout is configurable per instance (default 1 DGB).
◆ Prerequisites
A MagoFonte API endpoint running (local or hosted) — this page calls it
A DGB reward wallet address to receive block rewards
A free account on at least one provider below and your API token ready
Your DGB full node RPC host and port (mainnet default: 14022)
❱ Provider Setup Guide ❰
Complete step-by-step instructions for every supported provider. Select a provider tab to see how to create a free account, obtain your API token or deploy key, and paste it into the Launch form below. You only need to do this once per provider.
✈
Fly.io
Recommended · Global edge VMs · Free shared-cpu-1x allowance · Stratum TCP supported
Free 3 shared VMs30+ regionsDocker nativeCard required (no charge)
1
Create a Fly.io account
Go to fly.io/app/sign-up and register with your email. Fly requires a credit card for verification — you will not be charged on the free tier as long as you stay within 3 shared-cpu-1x VMs and 3 GB storage.
2
Install flyctl (CLI)
flyctl is the Fly.io command-line tool used to generate tokens. Install it with one command:
curl -L https://fly.io/install.sh | sh
On Windows (PowerShell):
iwr https://fly.io/install.ps1 -useb | iex
After install, add flyctl to your PATH if prompted. Verify with flyctl version.
3
Log in via CLI
flyctl auth login
This opens a browser window to authenticate. After login, your session is stored locally.
4
Generate a deploy token
Create a long-lived deploy token scoped to your organisation. This is what you paste into Lancia.
flyctl tokens create deploy -x 999999h
Copy the token printed to the terminal — it starts with FlyV1 fm2_. It will not be shown again.
Alternatively — get a token via the web dashboard: fly.io/user/personal_access_tokens → Create token → select Deploy Token.
5
Paste token into Lancia
Select Fly.io in the Provider dropdown in the Launch form below, then paste your token into the Provider API Token field. It is stored in browser memory only and cleared when you close the tab.
6
Choose a region
Pick the Fly region closest to your miners. Run flyctl platform regions to see all available regions and their latency from your location. Recommended: ord (Chicago) for North America, ams for Europe, sin for Asia-Pacific.
7
Free tier limits
Fly free allowance: 3× shared-cpu-1x VMs, 256 MB RAM each, 3 GB total volume storage, 160 GB outbound bandwidth/month. Stratum connections are TCP and do work on Fly — make sure to allocate a dedicated IP (flyctl ips allocate-v4) so miners can reach port 3333 from outside.
Note: shared-cpu-1x VMs sleep after 5 minutes of no HTTP traffic by default. For a mining pool, set min_machines_running = 1 in your fly.toml, or Lancia will handle this via the API at launch.

Railway
Easiest setup · GitHub-connected deploys · Free Hobby plan · US & EU regions
$5 free credit/monthGitHub deployEnv vars UINo static IP on free tier
1
Create a Railway account
Go to railway.app and sign up with GitHub (recommended) or email. The Hobby plan gives $5 of free compute credit per month — enough for a continuously running small pool instance.
2
Get your API token
In the Railway dashboard, click your avatar (top-right) → Account Settings → Tokens → Create Token. Name it lancia-deploy and click Create. Copy the token immediately — it is shown once.
Direct URL: railway.app/account/tokens
3
Paste token into Lancia
Select Railway in the Provider dropdown, then paste the token into the Provider API Token field. Lancia will use Railway's GraphQL API to provision a new service from the MagoFonte Docker image.
4
Set environment variables (optional manual path)
If you prefer to configure env vars manually instead of via Lancia: open your Railway project → select the service → Variables tab. Add each key-value pair from the Launch form. Required vars: POOL_ALGO, STRATUM_PORT, API_PORT, RPC_HOST, RPC_PORT, REWARD_ADDRESS.
5
Networking note
Railway free tier does not provide a static IPv4 address. Miners must connect via Railway's auto-generated public domain (e.g. pool-skein-01.up.railway.app:3333). TCP ports other than 80/443 require a paid plan or a Railway TCP Proxy — configure this under your service's Settings → Networking → TCP Proxy.
Tip: For Stratum mining, Railway TCP Proxy is the free workaround. Lancia will configure this automatically when the Railway provider is selected.

Render
Simple UI · Free web services · Auto-deploy from Docker · US Oregon & Frankfurt
Free web serviceDocker deployAuto TLSSleeps after 15 min idle
1
Create a Render account
Go to render.com and sign up with GitHub, GitLab, or email. No credit card required for the free tier. Free web services run 24/7 but sleep after 15 minutes of inactivity (no incoming HTTP requests). For a mining pool, you need to keep it alive — Lancia sends a keep-alive ping every 10 minutes automatically.
2
Get your API key
In the Render dashboard: click your avatar (top-right) → Account Settings → API Keys → Create API Key. Name it lancia and copy the key.
Direct URL: dashboard.render.com/u/settings#api-keys
3
Paste key into Lancia
Select Render in the Provider dropdown and paste your API key into the Provider API Token field. Lancia will call the Render API to create a new Web Service from the MagoFonte Docker image.
4
Environment variables
Lancia injects all required environment variables at deploy time via the Render API. To view or edit them manually after deploy: Render dashboard → your service → Environment tab. Changes trigger a redeploy automatically.
5
TCP / Stratum port limitation
Render free tier only exposes port 443 (HTTPS) publicly. Raw TCP (Stratum) requires a paid plan. As a workaround, Lancia can configure a WebSocket-to-TCP Stratum proxy on port 443 — miners using cgminer/sgminer with WS support can connect. Enable this with the env var STRATUM_WS_PROXY=true in the Advanced overrides field.
Recommendation: Use Fly.io or Oracle for Stratum pools. Use Render for the API / dashboard endpoint only.

Oracle Cloud Free Tier
Always Free · ARM VMs · 4 OCPUs + 24 GB RAM total · Full root access · Best raw performance
Always Free (no expiry)ARM Ampere A14 OCPUs freeSetup is more involved
1
Create an Oracle Cloud account
Go to oracle.com/cloud/free and click Start for free. You will need a credit card for identity verification — you are not charged. The Always Free tier never expires. Choose your home region carefully — it cannot be changed later. Recommended: us-ashburn-1 (East US) or eu-frankfurt-1.
2
Create an API signing key
Oracle uses RSA key-pair authentication for API access. In the OCI Console: click your profile icon (top-right) → User Settings → API Keys → Add API Key. Choose Generate API Key Pair, download both the private key (.pem file) and copy the configuration snippet shown.
The config snippet contains: user OCID, tenancy OCID, region, fingerprint, and key_file path. You need all of these for Lancia.
3
Paste credentials into Lancia
Select Oracle Cloud in the Provider dropdown. The token field accepts a JSON blob with all required Oracle credentials:
{"user":"ocid1.user.oc1..xxx","tenancy":"ocid1.tenancy.oc1..xxx","region":"us-ashburn-1","fingerprint":"aa:bb:cc:...","privateKey":"-----BEGIN RSA PRIVATE KEY-----\n...\n-----END RSA PRIVATE KEY-----"}
Paste the private key contents (from your downloaded .pem file) as a single-line string with \n for newlines, or use the multi-line textarea that appears when Oracle is selected.
4
Configure VCN security rules (open ports)
Oracle blocks all inbound traffic by default. You must open the Stratum port manually: OCI Console → Networking → Virtual Cloud Networks → your VCN → Security Lists → Add Ingress Rule. Set Source CIDR: 0.0.0.0/0, Protocol: TCP, Destination Port: 3333 (or your chosen Stratum port).
Also open port 3000 for the MagoFonte API if you want to reach it externally.
5
Free tier VM specs
Oracle Always Free ARM allocation: up to 4 OCPUs and 24 GB RAM split across Ampere A1 instances (e.g. 4× 1-OCPU/6-GB or 1× 4-OCPU/24-GB). You also get 2× AMD micro VMs (1/8 OCPU, 1 GB RAM). For MagoFonte, a 1-OCPU/6-GB ARM instance is more than sufficient for a production pool.

Custom / Self-Hosted
Full control · Any server with Docker · VPS, homelab, or bare metal · No provider lock-in
Any Docker hostFull TCP supportNo vendor limits
1
Prerequisites on your server
You need a server (VPS, homelab machine, or bare metal) with Docker installed. Minimum specs: 1 CPU core, 512 MB RAM, Ubuntu 20.04+ or Debian 11+.
curl -fsSL https://get.docker.com | sh
2
Deploy the MagoFonte Lancia agent
The Lancia agent is a lightweight HTTP server that runs on your machine and accepts Lancia API calls to start/stop/manage pool containers. Run it with:
docker run -d --name lancia-agent \ -p 4444:4444 \ -v /var/run/docker.sock:/var/run/docker.sock \ -e AGENT_TOKEN=your-secret-token \ ghcr.io/5mil/magofonte-lancia-agent:latest
Replace your-secret-token with any strong random string. This token is what you paste into Lancia as the Provider API Token.
3
Point Lancia at your agent
Select Custom / Self-hosted in the Provider dropdown. Two new fields will appear: Agent Endpoint (e.g. http://your-server-ip:4444) and Agent Token. Fill both in. All Lancia API calls will be proxied through your agent.
4
Open firewall ports
Ensure your server's firewall allows inbound TCP on your Stratum port (default 3333) and optionally the API port (3000). On UFW:
ufw allow 3333/tcp ufw allow 3000/tcp ufw reload
5
HTTPS / TLS (recommended)
For production, put the Lancia agent behind an NGINX reverse proxy with a Let's Encrypt certificate so the token is never sent in plaintext. A basic setup:
apt install nginx certbot python3-certbot-nginx -y certbot --nginx -d your-domain.com
Then proxy https://your-domain.com/api/ → http://localhost:4444/ in your nginx config.
